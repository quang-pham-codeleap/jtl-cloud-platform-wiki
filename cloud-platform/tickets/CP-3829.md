# CP-3829 — [FE] Update useAppsQuery to filter by installationStatus

**Jira:** https://jtl-software.atlassian.net/browse/CP-3829
**Component:** Core React Libraries
**Parent:** PRO-3410
**Source:** CERP-1150
**Status:** Open

---

## Instance

* All instances

## Problem to solve

* `useAppsQuery` has no `installationStatus` field in `AppsQueryVariables.filters` and doesn't pass it to `getApps`. Once CP-3828 ships, there's no way for callers to opt in to filter by installation status — causing stale uninstalled apps to surface in the ERP sidebar (CERP-1150).

## User Story

* n.a.

## Designs

* n.a.

## To do/actions

* Add `installationStatus` to `AppsQueryVariables.filters` in `useAppsQuery.ts`
* Forward `installationStatus` to `getApps` query params
* Update callers that should pass `installationStatus: 'Installed'` explicitly after CP-3828 deploys

## Acceptance Criteria

* `AppsQueryVariables.filters` includes `installationStatus`
* `useAppsQuery` forwards it to `getApps` when provided
* ERP sidebar no longer shows stale links after uninstall (validates CERP-1150)
* No regressions in existing callers

## Technical notes

* Blocked by CP-3828

## Test process

1. Uninstall an app
2. Call `useAppsQuery` without filter — verify uninstalled app is not returned
3. Call `useAppsQuery` with `installationStatus: 'Uninstalled'` — verify it is returned
4. Verify ERP sidebar no longer shows the stale link
