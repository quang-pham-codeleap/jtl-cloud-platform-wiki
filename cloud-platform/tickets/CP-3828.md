# CP-3828 — [BE] add installationStatus as parameters for the GET /apps endpoint

**Jira:** https://jtl-software.atlassian.net/browse/CP-3828
**Component:** App Service
**Parent:** PRO-3410
**Label:** CP_Services
**Source:** CERP-1150
**Status:** Open

---

## Instance

* All instances

## Problem to solve (must)

* `GET /apps` (`QueryAppInstancesQueryHandler.cs`) filters app instances only by `TenantId` (and optionally `promotionStatus` / search term) — there is no `InstallationStatus` filter. As a result, uninstalled app instances are returned alongside installed ones.
* This causes the ERP sidebar to display stale links for apps that have been uninstalled (source: CERP-1150).

## User Story

* As a consumer of the App Service API, I want `GET /apps` to return only `Installed` app instances by default, with an optional `installationStatus` query param for callers that need other statuses, so that uninstalled apps are not surfaced to the ERP sidebar.

## Designs

* n.a.

## To do/actions (must)

* In `QueryAppInstancesQueryHandler.cs`, add a default filter of `InstallationStatus = Installed` (mirroring `QueryAppInstancesCrossPartitionQueryHandler.cs` and `QueryTenantAppsQueryHandler.cs`)
* Expose `installationStatus` as an optional query parameter on `GET /apps` so callers can explicitly request other statuses when needed
* Update the OpenAPI/Swagger spec to document the new query parameter

## Acceptance Criteria (must)

* `GET /apps` without `installationStatus` returns only `Installed` app instances
* `GET /apps?installationStatus=Uninstalled` returns only uninstalled instances
* Invalid `installationStatus` values return `400`
* After uninstalling an app, the ERP sidebar no longer shows the stale link (validates CERP-1150)
* OpenAPI/Swagger spec is updated


## Test process

1. Install an app, then uninstall it
2. Call `GET /apps` — verify the uninstalled app is NOT returned
3. Call `GET /apps?installationStatus=Uninstalled` — verify the uninstalled app IS returned
4. Call `GET /apps?installationStatus=invalid` — verify `400` is returned
5. Verify ERP sidebar no longer shows the stale link after uninstall (validates CERP-1150)

## Description of the breaking changes (must)

* **Breaking change for clients relying on `GET /apps` returning all statuses.** Callers currently receiving uninstalled instances will no longer get them by default — they must pass `installationStatus=Uninstalled` explicitly.
