# CP-3819 — [Research] Classify and sort component inventory by migration category

**Jira:** https://jtl-software.atlassian.net/browse/CP-3819  
**Parent:** PRO-3045 — Component Library - DX Improvements (Documentation, Tokens, and Workflow) - Q2/26  
**Status:** Open

---

**Relevant Component:**

- All components in `jtl-platform-ui-react` and `jtl-platform-internal-react`

**Problem to solve (must):**

- We have no shared way to classify components. We need a tier model (Atoms → Molecules → Organisms → Templates) and a note on how each component is built (compositional vs. attribute-based).

**User Story / Jobs To Be Done (optional):**

- As a frontend developer, I want each component to have a tier and a composition style, so I can quickly judge its role and what to do with it during migration.

**Designs:**

- n.a.

**To do/actions (must):**

- Write a short definition for each tier:
  - (1) **Atoms** — smallest indivisible UI units (e.g. Button, Input, Icon)
  - (2) **Molecules** — simple combinations of atoms (e.g. FormField, SearchBar)
  - (3) **Organisms** — complex sections made of molecules/atoms (e.g. DataTable, Modal)
  - (4) **Templates** — full page-level layouts that arrange organisms
- Note the composition style per component:
  - **Compositional** — built via child components / slots
  - **Attribute-based** — driven entirely by props, no sub-components exposed
- Classify every component in both repos.
- Produce a reference table: component name, repo, tier, composition style.

**Acceptance Criteria (must):**

- Each tier has a written definition.
- Every component has an assigned tier and composition style.
- The reference table exists (Confluence or markdown).
- Migration work can be sequenced from the table.
